# v7 Composite Verify Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Behavior contracts + test specs, no reference code.

**Goal:** One call = one verdict. `verify` composes the existing machinery — invariant check across a standard viewport sweep, optional forced states, optional per-viewport snapshot diff — into a single PASS/FAIL answer for the agent's "is this OK?" moment, saving 3–6 round trips per verification.

### Task 1: `verify` — CLI command + MCP tool

**Files:** Create the composite in `src/core/matrix.ts` or a new `src/core/verify.ts` (implementer's judgment; one clear responsibility). Modify `src/cli.ts` (new command + USAGE), `src/mcp.ts` (seventh tool), README (command table + example). Tests: `test/cli.test.ts`, `test/mcp.test.ts`. Fixtures: reuse `fixtures/hover` and `fixtures/responsive` — no new fixtures needed.

**Contract:**
1. `bettercss verify <url> [--viewports LIST] [--hover S] [--focus S] [--active S] [--name NAME --dir DIR]` and MCP tool `verify` with the matching optional params. Default viewports when none given: `375x800,768x800,1280x800` (the standard sweep — documented in USAGE and the tool description).
2. Per viewport (input order, sequential): run invariants (with forced states when given, reusing v6's per-viewport forcing); if `name` is given, also diff against the per-viewport snapshot `<name>@WxH.tree` (or plain `<name>.tree` when the user passed a single explicit viewport — match the existing snapshot naming convention exactly: matrix runs use @WxH, so verify ALWAYS uses @WxH naming since verify is always a matrix, even with one viewport; document this).
3. Missing per-viewport snapshot when `name` was given → a note line (`note: no snapshot '<name>@WxH' — diff skipped for this viewport`), NOT a failure and NOT an error (agents may snapshot only some viewports).
4. Output: FIRST line is the verdict — `VERDICT: PASS` or `VERDICT: FAIL (<N> violations across <M> viewports<, K layout changes> )` — followed by the existing `[WxH]`-prefixed detail lines (violations with suspects, diff lines, busy-notes) and the existing summary line. Exit 1 on FAIL, 0 on PASS (CLI). Layout changes from diff count toward FAIL.
5. States respect the v5 exclusions: verify's snapshot-diff portion runs STATELESS (states apply to the check portion only — diffing a forced-state layout against a resting snapshot would always "fail"; document this in the tool description: "states affect the invariant check; the snapshot diff always compares the resting layout"). This requires the check extraction and diff extraction to be separate page loads per viewport when both states and name are given (2 pages per viewport worst case — acceptable; note it).
6. `snapshot`/`diff`/`check` and all other commands byte-identical (zero changes to their code paths).

**RED tests (must fail on current code — command/tool doesn't exist):**
- (a) CLI: `verify <responsive fixture>` (defaults) → exits 1, first line `VERDICT: FAIL`, contains `[375x800]`- or `[600-ish]`-prefixed overflow... (the responsive fixture overflows below 720px: with default sweep, 375x800 dirty, 1280x800 clean) — assert verdict line, a `[375x800]` violation, and `1280x800=clean` in the summary.
- (b) CLI: `verify <basic fixture>` → exits 0, first line `VERDICT: PASS`.
- (c) CLI: hover fixture with `--hover .cta --viewports 1280x800,600x800` → FAIL with `[1280x800]` bleed only (reuses v6 semantics through the composite).
- (d) CLI: snapshot the responsive fixture at `--viewports 1280x800` then `verify --viewports 1280x800 --name <n> --dir <tmp>` → PASS with no diff lines; then verify with a name that has NO snapshot files → PASS (if invariants clean) with the note line present.
- (e) MCP: `verify` tool listed (tool count 7) and returns the verdict-first output for the basic fixture.

**Steps:** RED tests → implement → full evidence run → commit `Added composite verify command and tool`.

## Self-Review
Verdict-first output is the contract's point (agents read line 1). The states-vs-diff separation (contract 5) is the one subtle design decision — resolved in favor of never producing a guaranteed-false diff, at the cost of a second page load per viewport in the combined case. Default sweep hardcodes three widths; making it configurable beyond --viewports is YAGNI.
